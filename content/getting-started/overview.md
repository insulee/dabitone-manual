# 전체 화면 구조

DabitOne 실행 시 보이는 화면 구성과 주요 UI 용어, 첫 사용자가 따라갈 표준 워크플로우를 한 페이지로 정리. 각 탭의 세부 컨트롤은 [[ui-reference/index|UI 레퍼런스]]에서.

## 화면 레이아웃

```
┌───────────────────────────────────────────────────────┐
│ 상단 상태 영역  (연결 상태·토스트 알림·제품 버전)     │
├──────┬────────────────────────────────────────────────┤
│ 사이드바 │                                            │
│          │     메인 콘텐츠 영역                       │
│ [통신]   │     (선택된 탭에 따라 바뀜)                │
│ [설정]   │                                            │
│ [전송]   │                                            │
│ [편집]   │                                            │
│ [고급]   │                                            │
│          │                                            │
│ [테마]   │                                            │
└──────┴────────────────────────────────────────────────┘
│ 하단 로그 영역  (지우기 버튼, 스크롤 이력)            │
└───────────────────────────────────────────────────────┘
```

## 사이드바 5 그룹

DabitOne의 가장 중요한 UI 개념. 왼쪽에 세로로 배열된 5개의 라디오 버튼이 **작업의 흐름 순서**를 나타냅니다.

| 순서 | 탭 | 역할 | 언제 쓰나 |
|------|-----|-----|---------|
| 1 | **[통신]** | 컨트롤러와 연결 | 매일 첫 작업 |
| 2 | **[설정]** | 화면 크기·색상 깊이·표출신호·폰트 | 설치 시 1회, 이후 거의 안 씀 |
| 3 | **[전송]** | 메시지·스케줄·밝기·배경 보내기 | 매일 콘텐츠 업데이트 시 |
| 4 | **[편집]** | 콘텐츠(텍스트·이미지·GIF·스케줄) 제작 | 콘텐츠 새로 만들 때 |
| 5 | **[고급]** | 펌웨어·로그·진단·시간 설정 | 문제 발생 시, 펌웨어 갱신 시 |

> [!info] 좌측 **[통신]**이 DabitOne 실행 시 기본 선택되어 있습니다. 연결이 성공해야 나머지 탭의 기능이 실제 컨트롤러에 반영됩니다.

## 첫 사용자 추천 워크플로우

처음 DabitOne을 쓰는 분을 위한 표준 순서. 각 단계 끝에 "투어 링크"를 따라가면 실제 클릭 스텝을 인터랙티브로 체험할 수 있습니다.

### Day 0 — 설치·연결 (10~20분)

1. [[getting-started/install|설치]] (5분)
2. 컨트롤러 하드웨어 배선 (Serial 케이블 또는 Ethernet)
3. **[통신] 탭**에서 연결 방식 선택 → 설정 입력 → [연결 테스트]
4. 녹색 "연결 성공" 확인 → [/tour/quickstart/01-first-connection/](/tour/quickstart/01-first-connection/) 투어로 세부 체험

### Day 0 — 화면 구성 (5~10분)

1. **[설정] 탭**에서 전광판 실물 사양(가로·세로 모듈 수, BPP, 배열) 입력
2. [전송] 버튼으로 컨트롤러에 화면 구성 적용
3. 필요하면 표출신호·폰트 설정 → [/tour/quickstart/02-screen-size/](/tour/quickstart/02-screen-size/)

### Day 0 — 첫 메시지 (5분)

1. **[편집] 탭**에서 "콘텐츠 편집"으로 텍스트 입력
2. 글꼴·색상·효과 선택 → 미리보기 확인 → 저장
3. **[전송] 탭**에서 메시지 선택 → [전송] → 실물 전광판에서 표출 확인
4. 자세히: [/tour/quickstart/03-send-message/](/tour/quickstart/03-send-message/)

### Day 1+ — 일상 운영

- 매일: [편집]에서 당일 콘텐츠 제작 → [전송]으로 송출
- 주간 단위: 스케줄(PLA) 편집 → [/tour/quickstart/06-schedule-pla/](/tour/quickstart/06-schedule-pla/)
- 월간 단위: 배경(BGP) 교체, 펌웨어 점검 → [/tour/quickstart/08-firmware/](/tour/quickstart/08-firmware/)

## 주요 UI 용어

### 그룹박스 (GroupBox)

회색 테두리로 둘러싸인 컨트롤 묶음. 헤더에 제목(또는 라디오 버튼)이 있고 내부에 관련 컨트롤이 모여 있습니다. 예: 통신 탭의 `Serial`, `Client TCP/IP` 그룹박스.

### 라디오 버튼

여럿 중 하나만 선택 가능. 그룹박스 헤더에 들어가 있을 때는 해당 그룹의 활성화 상태를 나타냅니다.

### 드롭다운 (ComboBox)

클릭하면 목록이 펼쳐지는 선택 컨트롤. 예: `포트`, `속도`, `응답시간`, `BPP`, `Color Order`.

### 입력 필드 (TextBox)

문자·숫자를 타이핑하는 컨트롤. 예: `IP Address`, `IP Port`, `세로`, `가로`.

### 체크박스 (CheckBox)

켜거나 끄는 토글. 예: `RS-485 Address`, `KeepOpen`, 폰트그룹 활성화.

### 토스트 알림

화면 하단 또는 우측 하단에 잠깐 뜨는 색상 알림.

- 🟢 **녹색** = 성공 (예: "연결 테스트 성공")
- 🟡 **노란색** = 경고·응답 없음
- 🔴 **빨간색** = 실패·오류

### 상태 영역

화면 상단에 고정된 영역. 현재 연결 상태와 프로토콜을 항상 표시합니다.

- "연결됨 (Serial)" 녹색
- "연결됨 (Client TCP/IP)" 녹색
- "연결 안됨" 빨간색

### 로그 영역

화면 하단의 시간순 텍스트 이력. 모든 전송·에러·상태 변화가 남음. `[지우기]` 버튼으로 초기화. 지원 요청 시 스크린샷 복사에 유용.

## 데이터 저장 위치

DabitOne이 관리하는 파일들의 실제 경로:

| 경로 | 내용 |
|---|---|
| `C:\Program Files\DabitOne\` | 실행 파일 (제거 시 삭제) |
| `C:\Users\<사용자>\Documents\DabitOne\Data\` | DAT·ANI·PLA·BGP 프로젝트 (사용자 작업물) |
| `C:\Users\<사용자>\AppData\Roaming\DabitOne\` | 앱 설정·최근 사용·테마 |
| `C:\Users\<사용자>\AppData\Roaming\DabitOne\logs\` | 로그 파일 (문제 보고 시 첨부) |

제거 후에도 프로젝트 파일은 `Documents\DabitOne\Data\`에 남으므로 재설치해도 복원 가능.

## 탭 간 공통 작업

### 다크/라이트 테마

[[ui-reference/05-advanced/theme|고급 → 테마]]에서 전환 가능. 야간 작업 환경 또는 선호에 따라. 사이드바 하단의 테마 스위쳐로도 빠르게 변경.

### 설정 영속성

각 탭에서 입력하거나 변경한 값은 대부분 자동 저장되어 **다음 실행 시 복원**됩니다. 별도 "저장" 버튼 없음. 단, 편집 탭의 콘텐츠는 명시적으로 [저장] 필요.

### 단축키

| 키 | 동작 |
|----|------|
| `Ctrl+Enter` | 현재 탭의 주요 액션 (예: 전송, 연결 테스트) |
| `Ctrl+S` | 편집 중인 콘텐츠 저장 |
| `F5` | 연결 상태 새로고침 |
| `Ctrl+L` | 로그 영역으로 포커스 이동 |
| `Ctrl+,` | 테마 토글 (다크/라이트) |
| `←` / `→` | 투어 페이지에서 스텝 이동 |

### 로그 복사 (지원 요청 시)

문제 재현 후 로그 영역 우클릭 → **전체 복사** → 이메일·지원 티켓에 첨부.
