# 연결이 안 될 때

> [!bug] 이런 경우에 읽으세요
> - **[연결 테스트]** 클릭 후 "응답 없음" 노란색 토스트
> - "연결 테스트 실패" 빨간색 토스트 (에러 메시지 포함)
> - 상단 상태 영역이 계속 "연결 안됨" 빨간색
> - COM 포트 드롭다운에 장비가 안 보임

## 먼저 확인 (5가지 체크)

- [ ] 컨트롤러 **전원**이 들어와 있다 (LED 점등·팬 회전 등으로 확인)
- [ ] **케이블** 결합이 단단하다 (흔들리거나 빠져 있지 않음)
- [ ] Serial 연결이면 **COM 포트**가 다른 프로그램에서 점유되지 않았다 (장치 관리자 확인)
- [ ] TCP/UDP이면 PC와 컨트롤러가 **같은 네트워크**(서브넷)에 있다
- [ ] 최근 **펌웨어 업데이트** 후 통신 설정이 바뀌지 않았는지

## 증상: "응답 없음" (노란색 토스트)

요청은 보냈지만 컨트롤러가 정해진 시간 내에 응답하지 않았다는 뜻.

### 가능한 원인

- **물리 계층 문제**: 케이블 단선, 단자대 +/- 반전, 커넥터 불량
- **Serial 속도 불일치**: 컨트롤러는 9600인데 DabitOne은 115200 등
- **응답 지연 초과**: 컨트롤러가 바쁘거나 느린 경우 기본 3초로 부족
- **네트워크 지연** (TCP): 같은 서브넷이지만 라우터 경유로 지연

### 조치

1. 통신 창 하단 `응답시간`을 `3초`에서 `5초` 또는 `6초`로 올려 재시도
2. Serial이면 **[속도 찾기]** 버튼으로 통신되는 속도 자동 탐색
3. TCP이면 [Ping] 버튼으로 네트워크 도달성 먼저 점검
4. 케이블을 뽑았다 다시 연결하고 재시도
5. 그래도 안 되면 컨트롤러 재부팅

## 증상: "연결 테스트 실패: ..." (빨간색 토스트)

구체적 에러가 메시지에 표시됨.

### "COM 포트를 열 수 없습니다"

- COM 포트를 다른 프로그램(PuTTY, 다른 DabitOne 인스턴스 등)이 점유 중
- **조치**: 점유 중인 프로그램 종료 후 재시도

### "Host unreachable / No route to host"

- IP가 네트워크에 존재하지 않거나 라우팅 불가
- **조치**: 컨트롤러 IP 재확인, 서브넷 마스크 확인, 라우터·스위치 동작 확인

### "Connection refused"

- IP는 맞지만 컨트롤러가 해당 포트에서 수신 대기하지 않음
- **조치**: 포트 번호 재확인 (기본 5000), 컨트롤러 쪽 서비스 구동 상태 확인

### "Timeout"

- 위 "응답 없음"과 유사. 같은 방법으로 대응.

## 증상: COM 포트가 드롭다운에 안 나옴

### 가능한 원인

- USB-to-Serial 어댑터 드라이버 미설치·문제
- Windows가 어댑터를 인식하지 못함

### 조치

1. 통신 창 **[장치관리자]** 버튼 클릭 → 포트(COM/LPT) 섹션 확인
2. 어댑터가 느낌표(!)로 표시되면 드라이버 재설치
3. 어댑터가 아예 안 보이면 USB 포트·케이블 교체 시도

## 증상: 상단 상태가 계속 "연결 안됨"

### 가능한 원인

- 연결 테스트는 성공했지만 지속 연결이 끊어짐
- KeepAlive 설정 문제

### 조치

- 통신 창 하단 `KeepOpen` 옵션이 있다면 체크 후 재연결
- 그래도 끊기면 [[troubleshooting/05-faq|FAQ]] 확인
