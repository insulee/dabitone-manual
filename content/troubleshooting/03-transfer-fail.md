# 전송이 실패할 때

> [!bug] 이런 경우에 읽으세요
> - 전송 진행률이 **중간에 멈추거나** 후퇴
> - **"CRC 오류"**, **"체크섬 불일치"** 메시지
> - 전송 도중 **타임아웃**으로 실패 처리
> - 전송 완료 후에도 **컨트롤러 화면이 업데이트되지 않음**
> - 같은 내용을 여러 번 시도해도 실패

## 먼저 확인

- [ ] [연결](/troubleshooting/01-connection)이 먼저 성공한 상태인지 확인
- [ ] 상단 상태 표시가 **"연결됨"** 녹색인지
- [ ] 전송 중 **케이블을 건드리거나 PC가 슬립 모드**로 들어가진 않았는지
- [ ] 컨트롤러 **저장 공간**이 충분한지 (너무 많은 파일 누적 시 가득 찰 수 있음)
- [ ] 전송 중 다른 프로그램이 **COM 포트·네트워크를 점유**하지 않는지

## 증상: 진행률이 중간에 멈춤

진행 바가 50%에서 더 이상 움직이지 않음.

### 가능한 원인

- **통신 안정성 문제**: 케이블·무선 링크 신호 약함
- **컨트롤러 응답 지연**: 다른 작업 처리 중 응답 늦어짐
- **대용량 파일**: 한 번에 전송하기 너무 큰 이미지·GIF·PLA

### 조치

1. **[재시도]** 버튼으로 실패 지점부터 이어 전송
2. 전송 안 되는 파일이 특정 1~2개라면 그 파일만 골라 개별 전송 시도
3. 파일 용량이 크면 **분할**하거나 **해상도 조정** 후 재시도
4. Serial이면 속도를 **낮춰** 재시도 (예: 115200 → 57600)
5. TCP이면 **응답시간**을 늘림 (3초 → 6초)

## 증상: "CRC 오류" 또는 "체크섬 불일치"

전송된 데이터가 중간에 훼손됐다는 의미.

### 가능한 원인

- **물리 계층 노이즈**: 케이블 접촉 불량, 전자기 간섭(EMI)
- **속도 너무 높음**: Serial 921600 등 고속에서 오류율 증가
- **RS-485 종단 저항 누락**

### 조치

1. 케이블을 **흔들어 보며** 단단히 고정, 필요 시 교체
2. Serial 속도를 낮춰 재시도
3. RS-485 환경이면 버스 끝의 **120Ω 종단 저항** 확인
4. 대형 모터·조명 등 **EMI 발생원**에서 케이블을 떨어뜨림

## 증상: 타임아웃

정해진 시간 내 응답을 못 받아 실패.

### 가능한 원인

- **응답시간 설정이 너무 짧음**
- **컨트롤러가 응답을 못 보냄** (연결 끊김, 펌웨어 프리즈)
- **네트워크 지연** (TCP/UDP, 라우터 경유 시)

### 조치

1. 통신 창 `응답시간`을 **5초 또는 6초**로 상향 후 재시도
2. 컨트롤러 **전원을 껐다 다시 켜서** 상태 초기화
3. [연결 문제](/troubleshooting/01-connection) 체크리스트도 확인

## 증상: 전송 완료됐는데 화면은 그대로

진행률은 100%인데 전광판 내용이 바뀌지 않음.

### 가능한 원인

- **컨트롤러 자동 새로고침이 꺼짐** 상태
- **스케줄(PLA)이 다른 항목을 재생** 중
- **배경 레이어(BGP)가 덮고 있음**

### 조치

1. 컨트롤러 재부팅해 다시 읽도록
2. [페이지 전송](/ui-reference/03-transfer/page)에서 **표시할 페이지 번호**를 명시적으로 변경
3. [배경](/ui-reference/03-transfer/background)이 전면을 덮고 있는지 확인

## 증상: 컨트롤러 저장공간 부족

"저장 공간 부족" 에러 메시지.

### 조치

1. 컨트롤러 관리 도구로 **기존 파일 정리**
2. 사용 안 하는 스케줄·배경·이미지 삭제
3. 여전히 부족하면 펌웨어/컨트롤러 용량 확장 옵션 확인
