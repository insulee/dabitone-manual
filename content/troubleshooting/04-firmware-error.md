# 펌웨어 오류

> [!bug] 이런 경우에 읽으세요
> - 펌웨어 **업데이트가 중간에 실패**함
> - 업데이트 후 컨트롤러가 **부팅을 반복**하거나 응답하지 않음
> - **펌웨어 정보 읽기 실패** 메시지
> - 업데이트 후 일부 기능이 **사라지거나 오작동**

> [!warning] **펌웨어 업데이트 중에는 절대 전원을 끊거나 케이블을 뽑지 마세요.** 중단 시 컨트롤러가 벽돌(부팅 불가)이 될 수 있습니다.

## 먼저 확인

- [ ] 다운로드한 .bin 파일이 **해당 컨트롤러 모델용**인지 확인
- [ ] 파일 **무결성 검증** 통과 (DabitOne이 로드 시 검증)
- [ ] 업데이트 직전 연결 상태가 **안정적**이었는지
- [ ] 업데이트 중 **다른 작업을 동시에 실행**하지 않았는지

## 증상: 업데이트 중 실패

진행률이 중간에 멈추고 에러 메시지.

### 가능한 원인

- **통신 단절**: 업데이트 중 케이블 흔들림·전원 튀는 현상
- **파일 손상**: 다운로드 중 일부 깨짐
- **모델 불일치**: 다른 모델용 펌웨어

### 조치

1. 컨트롤러 **전원을 끄고 30초 뒤 재부팅** (전해 방전 유도)
2. 재부팅 후 DabitOne에서 연결 재시도
3. 같은 .bin 파일을 **다시 다운로드**해 파일 손상 여부 점검
4. 업데이트 재실행 — 이번엔 PC 슬립 끄고 케이블 고정
5. 계속 실패하면 제조사 지원 문의 (복구 펌웨어 또는 하드웨어 리커버리 필요 가능)

## 증상: 부팅 반복 (boot loop)

컨트롤러가 켜지자마자 꺼지고 다시 켜지는 것을 무한 반복.

### 가능한 원인

- 업데이트 중단으로 **부트로더·펌웨어 일부 훼손**
- 모델 불일치 펌웨어 적용

### 조치

1. 제조사 지원팀 연락 — **복구 모드(recovery mode)** 진입 절차 안내 필요
2. 자가 복구 가능한 경우:
   - JTAG·UART 등 별도 포트를 통한 복구 펌웨어 플래시
   - 공장 초기화 점퍼·버튼이 있으면 제조사 지침에 따라 사용

> [!warning] 자가 복구는 전문 지식 없이 시도하면 하드웨어를 완전히 망가뜨릴 수 있습니다. 반드시 제조사 지원 안내를 따라 주세요.

## 증상: "펌웨어 정보 읽기 실패"

업데이트 전 펌웨어 버전·정보를 읽지 못함.

### 가능한 원인

- 컨트롤러와 **연결이 불안정**
- 구버전 펌웨어에서 정보 쿼리 프로토콜이 달라 실패

### 조치

1. [[troubleshooting/01-connection|연결]] 먼저 안정화
2. 펌웨어 정보가 없어도 업데이트 자체는 진행 가능 (제조사 안내 확인)
3. 업데이트 후 정보 읽기가 정상화되면 OK

## 증상: 업데이트 후 기능 오작동

업데이트는 성공했으나 이후 화면·전송 등 문제.

### 가능한 원인

- **기존 설정값과 신버전 사이 호환성 이슈**
- 일부 섹션 **초기화 필요**

### 조치

1. [[ui-reference/02-settings/index|설정]] 탭에서 화면 크기·색상 깊이 **재입력·재전송**
2. 저장된 **스케줄(PLA)·배경(BGP)·메시지(DAT)** 다시 전송
3. 그래도 안 되면 제조사 지원 문의

## 예방 — 다음 업데이트 시

- **PC는 전원 연결·슬립 끄기**
- 업데이트 **직전 현 설정 백업** ([[ui-reference/05-advanced/index|고급]] 탭 → 설정 내보내기)
- **릴리즈 노트** 먼저 읽고 호환성 이슈 확인
