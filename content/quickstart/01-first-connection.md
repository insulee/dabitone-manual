# 컨트롤러 최초 연결

> [!abstract] TL;DR
> - DabitONe 실행 → 좌측 **[통신]** 탭(기본 선택)에서 통신 방식 하나를 고르고 설정 입력 → **[연결 테스트]** 클릭합니다.
> - 성공하면 **"연결 테스트 성공"** 녹색 토스트가 뜨고 상단 상태 영역에 **"연결됨"**이 녹색으로 표시됩니다.
> - 케이블·네트워크가 준비돼 있으면 3분 내외로 완료됩니다.

## 준비물

- 다빛솔루션 LED 컨트롤러(DABIT 계열)와 전원
- DabitONe 데스크톱 앱 설치 완료 — [[getting-started/install|설치 가이드]]
- 연결 방식에 맞는 물리 계층:
  - **Serial**: RS-232/RS-485 케이블, 시리얼 포트 내장 PC 또는 USB-to-Serial 어댑터
  - **Client TCP/IP**: 컨트롤러와 같은 네트워크의 이더넷/Wi-Fi, 컨트롤러 IP·포트
  - **UDP**: 동일 서브넷 이더넷, 컨트롤러 IP

## 1단계. 좌측 **[통신]** 탭 확인

앱을 실행하면 좌측 사이드바 맨 위의 **[통신]**이 기본으로 선택되어 있고, 우측에 `통신 설정` 창이 뜹니다. 다른 탭(설정·전송·편집·고급)에 가 있다면 **[통신]**을 클릭해 돌아옵니다.

우측 창 좌측 열에는 **Serial / Client TCP/IP / Server TCP/IP / UDP** 네 개의 그룹박스가 위에서 아래로 나열되어 있습니다. 각 그룹박스 헤더에 라디오 버튼이 있고, 선택된 방식 하나만 진하게 활성화되며 나머지는 흐리게 표시됩니다. 기본 선택은 **Serial**입니다.

## 2단계. 통신 방식 선택 + 설정

세 가지 중 현장 환경에 맞는 하나를 고릅니다. 잘 모를 때 판단 기준:

- 컨트롤러와 PC를 시리얼 케이블로 직결 → **Serial**
- 공유기·스위치를 통해 컨트롤러 IP를 알고 있음 → **Client TCP/IP**
- 브로드캐스트·단방향 송출이 필요 → **UDP**
- 컨트롤러 쪽에서 PC로 접속해 오는 구성 → **Server TCP/IP** (특수 환경, [[ui-reference/01-communication/tcp|UI 레퍼런스]] 참조)

### 2-A. Serial 연결

1. **`Serial` 그룹박스 헤더 라디오 버튼 클릭** — 그룹 내부 컨트롤이 활성화됩니다.
2. **`포트` 드롭다운에서 해당 COM 번호 선택** — 목록에 없으면 USB 어댑터 드라이버를 확인합니다. **[장치관리자]** 버튼을 누르면 Windows 장치관리자가 바로 열립니다.
3. **`속도` 드롭다운에서 통신 속도 선택** — 선택 가능한 값은 9600 / 19200 / 38400 / 57600 / **115200**(기본) / 230400 / 921600 입니다. 컨트롤러 펌웨어가 쓰는 속도와 반드시 일치해야 합니다. 모를 경우 **[속도 찾기]**를 눌러 자동 탐색합니다.
4. **RS-485 배선이라면 `RS-485 Address` 체크박스를 켜고 주소값 선택** — RS-232 직결이면 해제한 채 둡니다.

### 2-B. Client TCP/IP 연결

1. **`Client TCP/IP` 그룹박스 헤더 라디오 버튼 클릭**
2. **`IP Address`에 컨트롤러 IP 입력** — 기본 예시 `192.168.0.201`. 컨트롤러 라벨 또는 기존 설정 도구에서 실제 IP를 확인해 바꿉니다.
3. **`IP Port`에 포트 번호 입력** — 기본 `5000`. 컨트롤러 쪽 포트와 일치해야 합니다.
4. **(선택) `Ping Test` 줄의 [Ping] 버튼으로 네트워크 도달성 점검** — 응답이 없으면 IP·서브넷·방화벽을 먼저 확인합니다.

### 2-C. UDP 연결

1. **`UDP` 그룹박스 헤더 라디오 버튼 클릭**
2. **`IP Address`에 컨트롤러 IP 입력** — 기본 예시 `192.168.0.202`
3. **`IP Port`** 는 `5108`로 고정되어 있어 편집할 수 없습니다.

> [!note] BLE·MQTT·dbNet
> 통신 설정 창 하단의 **[BLE]**·**[MQTT]** 버튼은 별도 설정창을 엽니다. 우측 `dbNet` 패널은 네트워크에서 컨트롤러를 자동 검색하는 기능으로, 상세 설명은 [[ui-reference/01-communication/index|통신 UI 레퍼런스]]에 있습니다.

## 3단계. **[연결 테스트]** 실행

통신 설정 창 맨 아래 버튼 영역을 확인합니다.

1. **`응답시간` 드롭다운에서 대기 시간 설정** — 기본 **`3초`**. 네트워크가 느리거나 컨트롤러 응답이 늦으면 `5초`~`6초`로 올립니다.
2. **[연결 테스트] 클릭** — 현재 설정을 저장하고 컨트롤러에 에코(echo) 요청을 보냅니다.

### 체크포인트

- ✅ 하단 알림 영역에 **"연결 테스트 성공"** 녹색 토스트
- ✅ 앱 상단 상태 영역 표시가 **"연결됨 (Serial)"** 또는 **"연결됨 (Client TCP/IP)"** 등으로 녹색 전환
- ✅ 이후 **[설정]**·**[전송]**·**[편집]** 탭으로 이동해도 연결 상태가 유지됨

## 잘 안될 때

- **"연결 테스트 응답 없음" (노란색 토스트)** — 컨트롤러 전원, 케이블 체결을 먼저 점검합니다. Serial이면 `속도` 불일치, TCP/UDP이면 IP·포트 오입력이 가장 흔한 원인입니다. `응답시간`을 올려 다시 시도해 봅니다.
- **"연결 테스트 실패: ..." (빨간색 토스트)** — COM 포트를 다른 프로그램이 점유 중이거나, 네트워크가 다른 서브넷이거나, 컨트롤러 펌웨어가 응답하지 않는 상태일 수 있습니다.
- 증상별 상세 진단: [[troubleshooting/01-connection|연결이 안 될 때]]
