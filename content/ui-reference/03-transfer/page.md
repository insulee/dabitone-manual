# 페이지 전송

![페이지 메시지 UI](/assets/screens/reference/transfer-page.png)

> 경로: **[전송] → 페이지 섹션**

## 이 화면의 역할

컨트롤러는 **여러 페이지 슬롯**에 콘텐츠를 저장할 수 있습니다. 이 화면에서 현재 표시할 페이지를 전환하거나 슬롯을 초기화합니다.

## 기본 사용

1. `현재 표시 페이지` 확인
2. `표시할 페이지` 번호 선택
3. **[적용]** → 즉시 해당 페이지가 전광판에 표출

## 화면 구성

| 요소 | 유형 | 설명 |
|------|-----|-----|
| `현재 표시 페이지` | 표시 | 컨트롤러가 현재 표출 중인 페이지 번호 |
| `표시할 페이지` | ComboBox | 1 ~ N (컨트롤러 사양) |
| `순환 모드` | CheckBox | 모든 페이지를 주기적으로 순환 표시 |
| `순환 주기` | NumericUpDown | 순환 모드일 때 페이지당 표시 시간(초) |
| **[적용]** | Button | 선택 반영 |
| **[초기화]** | Button | 선택한 페이지 슬롯을 빈 상태로 |

## 주요 옵션

### 페이지 vs PLA의 차이

| 개념 | 페이지 | PLA |
|------|-------|-----|
| 저장 단위 | 단일 DAT/ANI | 여러 DAT/ANI + 스케줄 |
| 복잡도 | 단순 | 시간·조건 기반 순환 |
| 용도 | 특정 메시지 고정 표출 | 시간표 기반 플레이리스트 |

간단한 전광판은 페이지만 써도 충분. 시간대별로 다른 내용을 띄우려면 [PLA](/file-formats/pla) 사용.

### 순환 모드

체크 시 컨트롤러가 페이지 1→2→3→...→N→1 순으로 자동 전환. 각 페이지당 `순환 주기`만큼 표시.

### 초기화

> [!warning] 초기화하면 해당 슬롯의 **DAT·ANI 데이터가 삭제**됩니다. 되돌릴 수 없으니 주의.

## 설정 영속성

페이지 표시 설정은 컨트롤러에 즉시 저장. 재부팅 후에도 유지.
