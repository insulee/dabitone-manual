# 시간 설정

> 경로: **[고급] → 시간 섹션**

## 이 화면의 역할

컨트롤러는 시간 기반 스케줄([[file-formats/pla|PLA]], 자동 밝기, 켜기·끄기 등)을 **자체 실시간 시계(RTC)**로 실행합니다. 이 화면에서 RTC를 PC 시간 또는 NTP 서버와 동기화.

## 기본 사용 — PC 시간 동기화 (간단)

1. `현재 시각` 확인 — 컨트롤러에 저장된 현재 시간
2. **[PC 시간으로 설정]** 클릭 → DabitOne이 돌아가는 PC의 현재 시간을 컨트롤러에 전송
3. 확인 → 적용

## 기본 사용 — NTP 자동 동기화 (권장)

1. `NTP 사용` 체크
2. `NTP 서버` 주소 입력 (기본: `pool.ntp.org`)
3. `주기` 선택 (1일 / 1주 / 1개월)
4. **[저장]** → 컨트롤러가 주기적으로 자동 동기화

## 화면 구성

| 요소 | 유형 | 설명 |
|------|-----|-----|
| `현재 시각` | 표시 | 컨트롤러 RTC 읽어 실시간 표시 |
| `PC 시각` | 표시 | DabitOne PC의 현재 시간 (참고) |
| **[PC 시간으로 설정]** | Button | 즉시 PC 시간으로 덮어쓰기 |
| `NTP 사용` | CheckBox | 자동 동기화 활성화 |
| `NTP 서버` | TextBox | 서버 주소 |
| `주기` | ComboBox | 자동 동기화 주기 |
| `시간대(Timezone)` | ComboBox | UTC+9 (Seoul) 등 |

## 주요 옵션

### NTP vs PC 시간 동기화

- **NTP**: 컨트롤러가 인터넷에 접속 가능한 경우 **자동 지속 동기화**. 가장 안정적.
- **PC 수동**: 인터넷 없는 설치 현장에서 사용. 주기적으로 현장 방문해 재동기화 필요.

### 시간대

운영 지역의 시간대를 정확히 설정. 한국은 `UTC+9 (Asia/Seoul)`. 잘못 설정하면 스케줄 시각이 9시간 어긋남.

### DST (Daylight Saving Time)

한국은 DST 없음. 다른 지역 설치 시 해당 국가의 DST 적용 여부 확인.

## 제약

- NTP는 컨트롤러가 **인터넷 접속 가능**해야 함
- 오프라인 설치 환경에서는 **반드시 수동 PC 동기화** 필요 (최소 월 1회)
- RTC 배터리 수명 (보통 수 년)이 다하면 시간이 리셋됨

## 왜 시간 동기화가 중요한가

시간 기반 스케줄([[ui-reference/03-transfer/brightness|자동 밝기]], [[ui-reference/03-transfer/power|켜기·끄기]], [[file-formats/pla|PLA]])이 전부 컨트롤러 시계 기준으로 동작하기 때문. 시계가 어긋나면 **의도한 시간과 다른 시각에** 메시지가 뜨거나 전원이 제어됨.
