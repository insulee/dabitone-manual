# GIF 편집

> 경로: **[편집] → GIF 모드**

## 이 화면의 역할

**GIF 애니메이션을 직접 제작·편집**. 기존 GIF 불러와 프레임 단위로 고치거나 처음부터 만들 수 있음. 저장 시 [[file-formats/ani|ANI]] 포맷으로 변환되어 전송.

> [!info] 이 편집기는 DabitOne **신기능**입니다. 기존 DabitChe는 외부 GIF를 불러오기만 가능했습니다.

## 기본 사용

1. **[새로 만들기]** 또는 **[GIF 불러오기]**
2. 좌측 **타임라인**에서 프레임 순서 확인
3. 프레임 선택 → 가운데 캔버스에서 편집 (이미지 편집기와 동일 방식)
4. 각 프레임의 `딜레이` (표시 시간) 설정
5. `반복 횟수` 설정
6. **[저장]** → `.ANI` 또는 `.gif`로 내보내기

## 화면 구성

### 좌측: 타임라인 패널

| 요소 | 설명 |
|------|-----|
| 프레임 썸네일 | 현재 전체 프레임 목록 |
| **[+]** | 빈 프레임 추가 |
| **[복사]** | 선택 프레임 복사 |
| **[삭제]** | 선택 프레임 제거 |
| 드래그 재배치 | 프레임 순서 이동 |

### 가운데: 편집 캔버스

이미지 편집기와 동일 도구 — 그리기·텍스트 삽입·이미지 가져오기 등.

### 우측: 속성 패널

| 요소 | 설명 |
|------|-----|
| `프레임 딜레이` | 현재 프레임 표시 시간 (ms) |
| `반복 횟수` | 전체 애니메이션 반복 (0 = 무한) |
| `전환 효과` | 프레임 간 페이드·인스턴트 |
| `총 길이` | 전체 재생 시간 표시 |

### 하단: 재생 컨트롤

- ▶ 재생 / ⏸ 일시정지 / ⏹ 정지
- 재생 속도 배율 (0.5x, 1x, 2x)

## 주요 옵션

### 프레임 딜레이

프레임마다 다르게 설정 가능. 예: 중요한 프레임은 1000ms, 빠르게 지나갈 프레임은 100ms.

### 반복 횟수

- **0 (기본)**: 무한 반복
- **1**: 한 번만 재생 후 마지막 프레임 유지
- **N**: N번 반복

### 외부 GIF 불러오기

표준 GIF 파일을 불러오면 각 프레임이 **내부 편집 프레임**으로 변환됨. 이후 일반 이미지처럼 수정 가능.

### 용량 관리

프레임 수가 많고 해상도가 크면 ANI 파일 용량이 급증. 권장:

- 프레임 수 ≤ 50
- 해상도는 전광판 기준 (크게 할 필요 없음)

## 저장·내보내기

| 포맷 | 용도 |
|------|------|
| `.ANI` | 컨트롤러 전송용 (기본) |
| `.gif` | 다른 시스템·문서에서 사용 |
