# 분할 모드

> 경로: **[편집] → 분할 모드**

## 이 화면의 역할

전광판 화면을 **2~4개 섹션으로 나누고**, 각 섹션에 서로 다른 콘텐츠를 동시에 재생. 예: 상단에 공지, 하단에 날짜·시각 / 좌측에 이미지, 우측에 뉴스 티커 등.

## 기본 사용

1. `분할 레이아웃` 선택 (좌우 2분할 / 상하 2분할 / 4분할 등)
2. 각 섹션을 클릭 → 해당 섹션에 DAT/ANI 파일 배정
3. 각 섹션의 속도·효과는 [[ui-reference/03-transfer/section-effect|섹션 효과]] 화면에서
4. **[저장]** → 분할 설정 파일 생성 후 전송

## 화면 구성

### 레이아웃 선택

| 레이아웃 | 구조 |
|---------|------|
| 좌우 2분할 | `[좌][우]` |
| 상하 2분할 | `[상] / [하]` |
| 4분할 | `[좌상][우상] / [좌하][우하]` |
| 상단 + 하단 2분할 | `[상] / [좌하][우하]` |
| 사용자 정의 | 자유롭게 경계선 드래그 |

### 섹션 편집

각 섹션을 클릭하면 해당 영역이 **독립된 미니 편집기**로 열림:

- 섹션별로 DAT/ANI/텍스트 배정
- 섹션별 해상도 자동 계산 (전체 해상도 ÷ 분할 수)
- 섹션별 미리보기

### 속도·효과 (섹션 개별)

분할 모드에서는 각 섹션에 **서로 다른 효과·속도**를 줄 수 있음. 상세 설정은 [[ui-reference/03-transfer/section-effect|섹션 효과]] 참조.

## 주요 옵션

### 섹션 크기

기본은 균등 분할. 사용자 정의 모드에서 경계선을 드래그해 비율 조정 가능. 예: 상단 20%, 하단 80%.

### 섹션 간 경계

- **경계선 표시** 체크 시 섹션 사이 얇은 선 표시
- 체크 해제 시 경계 없이 매끄럽게 이어짐 (보통 권장)

### 각 섹션 가독성

작은 섹션엔 **글자 크기를 자동 축소**해야 가독성 유지. 너무 많은 분할은 가독성 저하.

## 제약

| 항목 | 제약 |
|------|-----|
| 최대 섹션 수 | 4 (컨트롤러 사양에 따라 더 가능) |
| 각 섹션 최소 해상도 | 모듈 1개 이상 |
| 총 용량 | 각 섹션 콘텐츠 합이 컨트롤러 메모리 내 |

## 설정 영속성

분할 레이아웃과 각 섹션 콘텐츠는 컨트롤러에 저장되어 재부팅 후에도 유지.
