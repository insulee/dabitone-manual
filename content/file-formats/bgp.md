# BGP

> **BGP** — **BackGround Playlist**. 전광판의 **배경 레이어**에 순환되는 이미지·애니메이션 스케줄. 전면의 [PLA](/file-formats/pla) 콘텐츠와 **동시에 재생**됩니다.

## 왜 별도 레이어?

브랜딩 배경·시간표·패턴 등을 **배경으로 항상 깔아 두고**, 그 위에 실시간 메시지를 띄우고 싶을 때 사용합니다.

```
[BGP 배경 이미지 A]
       ↓ (깊이 뒤)
[PLA 공지사항 1]
[PLA 공지사항 2]
       ↓ (바뀌며 순환)
[관객에게 보이는 화면]
  = 배경 A + 공지사항 N
```

## 어디서 만들어지나

[전송 → 배경 이미지](/03-transfer/background) 또는 [편집 → 스케줄 그리드](/04-editor/schedule-grid)에서 배경 모드로 생성.

PLA 생성과 유사하지만 **배경용 투명도·겹침 처리**가 추가됩니다.

## 구조

- **배경 항목 목록**: 배경으로 깔릴 DAT·ANI 파일 참조
- **항목별 설정**:
  - 시작 시각·종료 시각
  - 투명도 (0~100%) — 전면 메시지와 겹침 정도
  - 블렌드 모드 (normal·multiply·screen 등, 지원 시)

## 어디에 저장되나

```
기본: C:\Users\[사용자]\Documents\DabitOne\Data\
확장자: .BGP
```

## PLA와의 차이

| 항목 | PLA | BGP |
|------|-----|-----|
| 레이어 | 전면 | 배경 |
| 기본 투명도 | 100% (불투명) | 설정값 (보통 30~70%) |
| 용도 | 메시지·공지·안내 | 브랜드·패턴·배경 |
| 가독성 | 우선순위 높음 | 전면을 방해하지 않아야 함 |

> [!warning] BGP 투명도가 너무 높으면 PLA 메시지가 읽기 어려워집니다. 배경 디자인은 **여백·단색 위주**가 권장됩니다.

## 재생 동작

컨트롤러는 BGP와 PLA를 **독립적으로 재생**합니다:

- BGP가 10분 주기, PLA가 30초 주기면 각자 자기 주기로 순환
- 동기화되지 않음 — 의도된 동작

## 제약

- BGP는 선택 사항 — 설정 안 하면 단색 배경 또는 검정
- 컨트롤러가 **듀얼 레이어**를 지원해야 함 (최근 펌웨어만)

## 호환성

구형 펌웨어에서는 BGP가 무시될 수 있음. 컨트롤러 스펙·펌웨어 확인 필요.

## 관련 파일 형식

- [PLA](/file-formats/pla) — 전면 레이어 플레이리스트
- [DAT](/file-formats/dat) / [ANI](/file-formats/ani) — BGP가 참조하는 배경 이미지·애니
