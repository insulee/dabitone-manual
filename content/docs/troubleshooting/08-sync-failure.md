# 여러 컨트롤러 동기화 실패

큰 전광판을 여러 컨트롤러로 분할 구동하거나 옥외 간판 여러 대를 같은 메시지로 운영할 때, 컨트롤러마다 표시 내용·타이밍이 어긋나는 경우.

## 증상

- 좌측 컨트롤러는 메시지 A, 우측은 B를 표시 (같이 바뀌어야 하는데)
- 같은 PLA 스케줄인데 컨트롤러마다 다른 항목 재생 중
- 시간이 컨트롤러마다 다름 (초·분 단위)
- 일부 컨트롤러만 구버전 메시지 표출

## 원인

1. **전송 누락** — 일부 컨트롤러에 새 메시지·스케줄 전송 안 됨
2. **시계 동기 불일치** — 컨트롤러마다 RTC(실시간 시계)가 다른 시간
3. **네트워크 지연 차이** — TCP/UDP 환경에서 전송 타이밍 불균형
4. **펌웨어 버전 상이** — 일부 컨트롤러만 펌웨어 구버전
5. **Split Mode 설정 불일치** — 분할 모드(좌·우 반씩) 구성 어긋남

## 해결 절차

### 1. 전송 일괄 재확인

1. [전송] 탭에서 각 컨트롤러별 전송 이력·상태 확인
2. 실패·중단된 컨트롤러에 재전송
3. 로그 영역에 각 컨트롤러 성공 표시 확인

### 2. 시계 동기

- 각 컨트롤러에 [고급] 탭 → **시간설정** → [PC 시간 전송] 순차 실행
- 가능하면 같은 PC에서 연속 실행 (PC 시간이 일관되도록)
- 자주 어긋나면 NTP·RTC 배터리 점검 (하드웨어 이슈)

### 3. 펌웨어 버전 통일

1. [고급] 탭 → **펌웨어 업데이트** → [펌웨어 읽기] 각 컨트롤러 버전 확인
2. 다르면 최신으로 전체 일괄 업데이트 (투어 08 참조)
3. 재부팅 후 재연결

### 4. Split Mode 구성

- 편집기 [Split Mode] 또는 투어 04 편집기 상세 참조
- 좌측 컨트롤러는 영역 A, 우측은 영역 B로 할당
- 같은 PLA 스케줄이 각 영역에 다른 내용이 돼야 정상

### 5. 연결 순서 문제

- 모든 컨트롤러에 순차 전송하면 수 초 간격 생김
- 완전 동시 동기화는 어려움 — 수 초 이내면 허용
- 절대 동기화가 필요하면 마스터-슬레이브 구성 (dbNet) 검토
