# 표시가 깨질 때

> [!bug] 이런 경우에 읽으세요
> - 전광판 색상이 원본과 달리 **이상한 색조·밀림·띠**가 생김
> - 일부 모듈만 **까맣거나 이전 내용이 잔상**으로 남음
> - 글자·이미지가 **뒤집히거나 회전**되어 나옴
> - 모듈 경계 부분이 **이어지지 않고 끊어져** 보임
> - 가로·세로 비율이 맞지 않음 (눌리거나 늘어남)

## 먼저 확인

- [ ] **화면 크기 설정**이 실제 전광판 구성과 일치 ([설정](/docs/02-settings/overview) 탭)
- [ ] **색상 깊이**가 모듈이 지원하는 비트 수와 일치 (1비트 단색 / 3비트 RGB / 24비트 풀컬러)
- [ ] **모듈 배열**(가로·세로·회전) 옵션이 실물과 일치
- [ ] 최근 **펌웨어 업데이트** 후 설정값이 바뀌지 않았는지
- [ ] 모듈 **전원 공급**이 안정적인지 (전원 부족 시 색상·밝기 이상)

## 증상: 색상 밀림 (Color shift)

빨간색이 파랗게, 파란색이 녹색처럼 다른 색으로 보임.

### 가능한 원인

- **R/G/B 배선 순서 바뀜**: 모듈 교체·재배선 시 색상 라인 순서가 뒤바뀜
- **색상 깊이 불일치**: 1비트 모듈에 24비트 영상 전송 시
- **표출 신호 설정 오류**: `[설정] → 표출 신호`의 스캔 방식·refresh rate

### 조치

1. [설정](/docs/02-settings/overview) 탭에서 색상 깊이 확인·수정
2. `표출 신호` 섹션에서 스캔 방식(1/8, 1/16 등)을 모듈 사양에 맞게 변경
3. 그래도 안 되면 **모듈 쪽 배선**(R·G·B 라인) 점검 필요 — 설치 업체 연락

## 증상: 일부 모듈만 까맣거나 잔상

특정 모듈만 내용이 안 바뀌거나, 꺼진 듯 보임.

### 가능한 원인

- **모듈 간 신호 케이블 단선**: HUB 포트 연결 불량
- **모듈 전원 부족**: 전원 공급 장치(SMPS) 용량 초과
- **개별 모듈 고장**

### 조치

1. 문제 모듈의 **데이터·전원 케이블** 결합 재확인
2. 해당 모듈을 다른 위치로 옮겨 본다 → 위치별 문제면 HUB/케이블, 모듈 자체면 교체
3. 전원 표시등(LED)이 정상 점등하는지 확인

## 증상: 글자·이미지가 뒤집히거나 회전

표출 내용이 거꾸로 또는 옆으로 뒤집혀 나옴.

### 가능한 원인

- **모듈 배열 방향** 설정이 실물과 다름 (Normal / 180도 / 가로·세로 뒤바뀜)
- 모듈 장착 방향 자체가 잘못됨

### 조치

1. [설정](/docs/02-settings/overview) 탭의 **배열 방향** 옵션을 회전 상태에 맞게 변경
2. 여러 모듈이 섞여 있으면 각 모듈 방향 확인 필요

## 증상: 모듈 경계가 끊어짐

모듈 단위로 이미지가 어긋나 잘리거나 겹침.

### 가능한 원인

- **가로·세로 모듈 수** 입력 오류 (예: 실물 8×2인데 4×4로 설정)
- **모듈 해상도** 입력 오류 (예: 실물 32×16인데 16×16으로 설정)

### 조치

1. 실물 모듈을 세어 가로·세로 모듈 수 재입력
2. 모듈 라벨에서 픽셀 해상도 확인 → 해당 값으로 입력
3. 전광판이 여러 모듈 그룹으로 나뉘어 있으면 분할 모드([편집](/docs/04-editor/split-mode) 탭)도 확인

## 증상: 밝기가 고르지 않음

한쪽은 밝고 다른 쪽은 어두운 등 밝기 편차.

### 가능한 원인

- 시간대별 자동 밝기 스케줄 설정
- 개별 모듈 밝기 불일치 (캘리브레이션 필요)

### 조치

1. [전송](/docs/03-transfer/brightness) 탭 **밝기 설정**에서 현재 밝기 레벨 확인
2. 자동 스케줄이 이상한 값을 적용 중이면 수동 재설정
