# 설치

> [!abstract] TL;DR
> - 다빛솔루션 공식 사이트에서 DabitOne 설치 파일 다운로드
> - **기존 DabitChe와 별도 설치**되어 공존 가능
> - 설치 후 첫 실행 시 좌측 **[통신]** 탭에서 바로 연결 시도

## 시스템 요구사항

| 항목 | 최소 | 권장 |
|------|-----|-----|
| OS | Windows 10 (64비트) | Windows 11 |
| 메모리 | 4 GB | 8 GB 이상 |
| 디스크 | 500 MB 여유공간 | 2 GB |
| 디스플레이 | 1366 × 768 | 1920 × 1080 |
| .NET | 자동 설치 | — |
| USB | USB 2.0+ (Serial 연결 시) | — |
| 네트워크 | 100 Mbps (TCP/UDP 연결 시) | 1 Gbps |

> [!warning] Windows 7·8은 지원하지 않습니다. 해당 환경이라면 기존 DabitChe를 유지하시거나 OS 업그레이드가 필요합니다.

## 다운로드

1. **[다빛솔루션 공식 사이트](https://www.dabitsol.com)** 또는 구매처에서 안내받은 URL에서 **DabitOne 설치 파일** 다운로드
2. 다운로드 파일명: `DabitOne-Setup-x.x.x.exe`
3. 파일 크기: 약 150 MB (변동 가능)
4. 다운로드 후 파일 속성 → **디지털 서명** 탭에서 "다빛솔루션" 서명 확인 (보안 검증)

## 설치 절차

1. 다운로드한 `.exe`를 **우클릭 → 관리자 권한으로 실행**
2. Windows 보안 경고가 뜨면 **[추가 정보] → [실행]** (코드 서명된 설치 파일)
3. 설치 마법사 안내에 따라 진행:
   - 설치 경로 (기본: `C:\Program Files\DabitOne\`)
   - 시작 메뉴 바로가기
   - 바탕화면 아이콘 (선택)
   - 파일 연결 (`.DAT`·`.ANI`·`.PLA` 등 DabitOne으로 열기)
4. "설치 완료" 화면에서 **[DabitOne 실행]** 체크 상태로 종료
5. 처음 실행 시 Windows Defender 방화벽 허용 팝업이 뜨면 **[허용]**

### 기존 DabitChe와의 관계

- **별도 설치**되어 공존 가능 — 예전 버전 그대로 유지되며 DabitOne은 `C:\Program Files\DabitOne\`에 새로 설치됩니다
- 양쪽 모두 **기존 프로젝트 파일**을 읽을 수 있으므로 익숙해질 때까지 병행 사용 가능
- 완전히 이관 후 DabitChe는 제어판에서 삭제하셔도 됩니다

## 첫 실행

1. 바탕화면 또는 시작 메뉴에서 **DabitOne** 아이콘 클릭
2. 실행되면 좌측 사이드바 맨 위의 **[통신]** 탭이 기본 선택된 상태로 뜹니다
3. 여기서부터는 → [/quickstart/01-connect/](/quickstart/01-connect/) 투어가 안내

### 설치 검증 체크리스트

설치가 올바르게 됐는지 5분 안에 점검:

- [ ] 바탕화면·시작 메뉴에 DabitOne 아이콘 보임
- [ ] DabitOne 실행 시 스플래시 → 메인 창 진입 (10초 이내)
- [ ] 상단 제목에 버전 표기 (`DabitOne V1.x.x`)
- [ ] 좌측 사이드바에 5개 아이콘 (통신·설정·전송·편집·고급) 보임
- [ ] 하단 로그 영역 정상 (에러 메시지 없음)
- [ ] [고급] 탭 → 정보 → 라이선스·버전 조회 가능

하나라도 실패 시 아래 "자주 겪는 설치 오류" 확인.

## 자주 겪는 설치 오류

### "Windows에서 이 앱을 보호했습니다" 팝업

- **원인**: Windows SmartScreen이 서명된 앱이어도 배포 빈도가 낮으면 경고
- **해결**: 팝업 하단 **[추가 정보] → [실행]** 클릭. 이후 재실행 시엔 안 뜸

### ".NET Desktop Runtime이 필요합니다" 오류

- **원인**: DabitOne은 .NET 8을 번들로 포함하는데, 보안 프로그램이 번들 파일을 격리한 경우 발생
- **해결**: 
  1. 설치 파일 재다운로드
  2. 백신·보안 프로그램에서 `C:\Program Files\DabitOne\` 폴더 예외 등록
  3. 재설치

### "다른 인스턴스가 이미 실행 중입니다"

- **원인**: 이전 DabitOne 인스턴스가 비정상 종료 후 프로세스 남음
- **해결**: 작업 관리자에서 `DabitOne.exe` 프로세스 종료 후 재실행

### COM 포트가 장치 관리자에 안 보임

- **원인**: USB-to-Serial 어댑터 드라이버 미설치
- **해결**: 어댑터 제조사(FTDI·Prolific 등)의 Windows 드라이버 수동 설치. DabitOne 재시작

### 설치 중간 멈춤·진행률 고정

- **원인**: 보안 프로그램이 MSI 서브 프로세스 차단
- **해결**: 설치 취소 → 보안 프로그램 일시 중지 → 재설치 → 설치 후 보안 프로그램 재시작

## 자동 업데이트

- 앱 시작 시 신버전 확인 (인터넷 연결 시)
- 수동 확인: **[고급] 탭 → 정보 → [업데이트 확인]**
- 업데이트 파일이 있으면 다운로드·실행 안내
- 업데이트 중에는 현재 작업 저장 후 진행 권장

## 오프라인 설치

네트워크가 없는 환경에서도 설치 가능:
1. 인터넷 연결된 PC에서 설치 파일 다운로드 (약 150 MB)
2. USB로 대상 PC에 복사
3. 동일 절차로 설치 (.NET Runtime 번들 포함이라 네트워크 불필요)
4. 업데이트는 수동으로 새 설치 파일 재배포

## 제거

제어판 → 앱 → DabitOne → 제거. 사용자 설정·프로젝트 파일은 `C:\Users\<사용자>\Documents\DabitOne\`에 별도 보관되어 제거해도 안전.

완전 삭제를 원하면 위 폴더도 수동 삭제.

### 설정·데이터 백업

제거 전 백업하고 싶다면:
- `C:\Users\<사용자>\Documents\DabitOne\Data\` — DAT·ANI·PLA·BGP 프로젝트 파일
- `C:\Users\<사용자>\AppData\Roaming\DabitOne\` — 앱 설정, 최근 사용 목록, 테마

이 두 폴더를 통째로 복사해두면 재설치 후 동일 폴더에 붙여넣어 복원 가능.
